# 🧠 ForgeLoopAI 思考与迭代指南 (Thinking Document)

本文档记录了 ForgeLoopAI (v1.0) 诞生时的设计哲学、当前系统的优劣势分析，以及未来长期的迭代方向。它是我们持续打磨这个工具的思想灯塔。

---

## 1. 为什么要做 ForgeLoopAI？

在当前大模型编程工具（如 Cursor、Trae、Windsurf 等）爆发的背景下，大家都在卷“模型生成能力”和“IDE 智能补全”。但当我们用这些工具去开发十万甚至百万行级别的重型项目（如 Apache Doris）时，会发现 AI 最大的短板**不是写不出代码，而是缺乏“工程纪律”和“极易丢失上下文”**。

具体痛点如下：
- 🤯 **记忆力崩塌**：复杂的构建、部署环境（几十行的 `cmake` 或 `docker compose` 指令）每次重开对话都需要反复教给 AI。
- 🔄 **失控的死循环**：AI 遇到编译报错时，容易陷入“盲目修改 -> 报错 -> 再改”的死循环，直到把 Token 上下文窗口彻底撑爆。
- 🗑️ **状态不可追溯**：挂机跑了一晚上，第二天完全不知道 AI 解决了什么问题、引入了哪些新 Bug、消耗了多少成本。

ForgeLoopAI 的诞生就是为了填补这块空白：**给 IDE 里的 Solo Coder 外挂一个拥有“单轮强约束”与“本地长时记忆”的大脑**。

---

## 2. 当前版本 (v1.0) 的核心亮点

### 🎯 定位精准：Meta-Agent Controller
我们没有去重复造轮子（比如自己写一个调用大模型 API 的脚本，或者搞一套沉重的 CI/CD 引擎），而是承认了“IDE 自带的 Agent 已经足够强大”。ForgeLoopAI 选择了做它们的**外围控制器和上下文拼装器**，这种“借力打力”的思路极其轻量且高效。

### 🛑 强约束单轮闭环
在生成的 Prompt 里，我们强行切断了 AI 的无限循环欲，规定它：
**“跑完一圈（编译->部署->测试->修复）必须停下，并交出 JSON 战况报告。”**
这不仅控制了 Token 成本，还让人类开发者（我们）在每一轮之间重新拿回了掌控权。

### 🔪 大道至简的设计哲学
抛弃了数据库和状态机，纯粹依靠本地文件系统（`config.json` + `round_N.json`）来流转状态。
- 环境坏了？直接删掉项目文件夹重来。
- 想微调历史上下文？直接用记事本修改 JSON。
这种遵循 Unix 哲学的工具往往生命力最强，认知负担极低。

## 3. 局限性与隐患 (亟待优化的技术债)

### ⚠️ 半自动化的体力活
当前的工作流是：`敲命令生成 -> 复制 Markdown -> 粘贴给 Trae -> Trae 运行 -> 敲命令看结果`。
虽然比手写强了一万倍，但“人工搬运 Prompt”依然打断了沉浸式的开发体验。

### ⚠️ 容错能力 (Fault Tolerance) 较弱
我们高度依赖大模型的“指令遵循”能力，要求 Trae 必须把结果写成纯正的 JSON 文件。
如果大模型某次“发神经”：
- 少写了一个大括号
- 把 JSON 打印在了对话框里而不是写进文件
`fl push` 的下一轮解析就会出问题。系统目前对异常结果的容错与纠偏机制还不够健壮。

### ⚠️ 缺乏代码物理快照
虽然工具记录了每轮的 Bug 和 Fix，但如果第 3 轮 AI 把代码彻底改崩了，工具本身并没有提供“一键回滚代码到第 2 轮”的能力（目前仍需依赖用户自己用 Git 管理或 IDE 的 Local History）。

## 4. 未来的想象空间 (Roadmap)

在未来的迭代中，我们可以围绕以下几个方向持续深化 ForgeLoopAI 的价值：

### 阶段一：增强鲁棒性与体验 (v1.1)
- **智能容错解析**：当检测不到标准的 JSON 文件时，尝试从剪贴板或模糊文本中正则提取 `status` 和 `bugs_found` 等关键信息，自动兜底纠正。
- **自定义 Prompt 模版**：允许用户在 `config.json` 中覆盖默认的 Prompt 话术，以适应不同性格的 Agent（比如专门调教一个给 Cursor 用的模版）。

### 阶段二：自动化闭环加深 (v2.0)
- **代码状态快照 (Snapshot)**：在每次 `push` 推进下一轮之前，自动为指定的代码目录打一个轻量级的 Git Commit 或本地备份，一旦某轮失败，支持 `fl rollback <round>` 一键恢复。
- **自动读取终端报错**：不需要 AI 去解释遇到了什么 Bug，工具自动截取上一轮的终端标准错误输出（stderr）并注入到下一轮的 Prompt 中。

### 阶段三：终极形态 (v3.0)
- **IDE 原生插件化**：将 CLI 演进为一个 VS Code / Trae 的扩展。一键点击侧边栏的“开启下一轮”，自动将生成的 Prompt 发送给内置的 AI 对话框，彻底消灭“复制粘贴”。
- **多租户/云端看板**：如果团队都在用，可以将本地的 `history/` 同步到轻量级的云端（如 SQLite/PostgreSQL），并在 Web 端展示所有人、所有项目的 AI 填坑进度。
